# ADR 0006: GDPR

## Status
Proposed | ~~Accepted~~ | ~~Superseded~~

## Context

PII is needed as users need to create accounts for service.
PII can be input by users in tasks.

There are various GDPR rights that need to be serviced.

## Decision

Identity verification process is currently manual and will be defined later as 1 month deadline is paused on verification.
Services for GDPR rights Articles 15-20 will be provisioned.
Right to be Informed is a frontend concern under assumption there is no change to data usage.
Article 21 and 22 out of scope.

All supplied data will need to be evaluated for PII on requests due to admin visability on all data.

## Considerations

Are we a data controller or data processor?
- Requirements fit data processor

SAR and right to amendment and right to be forgotten:
How is PII going to be identified?

How can a person do a SAR?

Am I identifying PII on data in the application database or a separate database optimised for this?

There is 1 month for a SAR.

The 1 month period is paused when identify is being verified.
